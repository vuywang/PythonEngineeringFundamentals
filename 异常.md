# 1. 什么是异常？🤔

你先把“异常”理解成：

> **程序运行过程中发生的错误**

比如：

```python
print(1 / 0)
```

运行后会报错：

```python
ZeroDivisionError: division by zero
```

因为 0 不能当除数，所以程序运行到这里出问题了。

这就是异常。

---

# 2. 异常和语法错误有什么区别？📌

这个要分清。

## 语法错误

代码根本不符合 Python 语法，程序连运行都跑不起来。

比如：

```python
if 3 > 2
    print("yes")
```

这里少了冒号 `:`，这叫 **语法错误**。

---

## 异常

语法没问题，程序也能开始运行，**但是运行到某一步时出错了**。

比如：

```python
a = 10
b = 0
print(a / b)
```

语法对，但运行时报错。

所以你可以这样记：

> **语法错误：代码写错了，程序启动不了**
> **异常：程序运行过程中出了问题**

---

# 3. 常见异常有哪些？👀

你不用一上来全记住，但要先认识几个最常见的。

---

## 3.1 `ZeroDivisionError`

除零错误

```python
print(10 / 0)
```

报错：

```python
ZeroDivisionError
```

---

## 3.2 `NameError`

变量没定义

```python
print(age)
```

如果 `age` 没定义，就会报：

```python
NameError
```

---

## 3.3 `TypeError`

类型不对

```python
print("10" + 5)
```

字符串和整数不能直接相加，会报：

```python
TypeError
```

---

## 3.4 `ValueError`

值不合法

```python
int("abc")
```

`"abc"` 不是合法整数，会报：

```python
ValueError
```

---

## 3.5 `IndexError`

下标越界

```python
nums = [1, 2, 3]
print(nums[5])
```

列表只有 3 个元素，没有第 6 个，会报：

```python
IndexError
```

---

## 3.6 `KeyError`

字典里没有这个键

```python
info = {"name": "张三"}
print(info["age"])
```

没有 `age` 这个 key，会报：

```python
KeyError
```

---

## 3.7 `AttributeError`

对象没有这个属性或方法

```python
num = 10
num.append(1)
```

整数没有 `append()` 方法，会报：

```python
AttributeError
```

---

## 3.8 `FileNotFoundError`

文件不存在

```python
open("abc.txt", "r")
```

如果文件不存在，会报：

```python
FileNotFoundError
```

# 4. 程序一旦出现异常会怎么样？💥

默认情况下：

> **程序会立刻中断**

比如：

```python
print("开始")
print(10 / 0)
print("结束")
```

运行结果：

```python
开始
ZeroDivisionError: division by zero
```

你会发现：

* `"开始"` 打印了
* `"结束"` 没打印

因为程序在出错那一行就停了。

# 5. 为什么要学异常处理？🛠️

因为现实开发里，有些错误是“可能发生”的，但你不希望程序直接崩掉。

比如：

* 用户输入了错误内容
* 文件不存在
* 网络请求失败
* 数据格式不对
* 下标越界

这些情况，如果不处理，程序就直接爆了 😵

所以我们要学：

> **try...except**

---

# 6. 最基本的异常处理：`try...except` ⭐

格式：

```python
try:
    可能出错的代码
except:
    出错后执行的代码
```

例子：

```python
try:
    print(10 / 0)
except:
    print("出错了")
```

输出：

```python
出错了
```

程序不会直接崩，而是进入 `except`。

---

# 7. `try` 和 `except` 到底怎么执行？🔍

看这个：

```python
try:
    print("步骤1")
    print(10 / 0)
    print("步骤2")
except:
    print("出错了")
```

执行过程是：

### 第一步

进入 `try`

### 第二步

打印：

```python
步骤1
```

### 第三步

执行 `10 / 0`，发生异常

### 第四步

`try` 里后面的代码不再执行
所以 `"步骤2"` 不会打印

### 第五步

跳到 `except`

打印：

```python
出错了
```

---

# 8. 为什么不建议只写裸 `except`？⚠️

虽然这样写可以：

```python
try:
    ...
except:
    ...
```

但一般不推荐。

因为它会“什么异常都接住”，你反而不知道到底出了什么问题。

更推荐这样写：

```python
try:
    print(10 / 0)
except ZeroDivisionError:
    print("除数不能为0")
```

这样更清楚。

# 9. 捕获指定异常 ✅

```python
try:
    num = int("abc")
except ValueError:
    print("输入的不是合法整数")
```

输出：

```python
输入的不是合法整数
```

这说明：

> `except ValueError` 只处理 `ValueError` 这种异常

# 10. 捕获多个异常 🌟

有时候一段代码可能出现多种错误。

比如：

```python
try:
    num = int(input("请输入一个数字："))
    result = 10 / num
    print(result)
except ValueError:
    print("请输入合法数字")

    print("除数不能为0")
```

这里：

* 输入 `"abc"` → `ValueError`
* 输入 `0` → `ZeroDivisionError`

# 12. `as`：拿到异常对象 🧠

有时候你不只想知道“出错了”，还想知道“具体错什么”。

这时可以这样：

```python
try:
    print(10 / 0)
except ZeroDivisionError as e:
    print("发生异常：", e)
```

输出：

```python
发生异常： division by zero
```

这里：

* `e` 就是异常对象
* 里面带着异常信息

# 13. 这有什么用？🤔

很有用，因为你可以打印真实错误信息。

比如：

```python
try:
    num = int("abc")
except ValueError as e:
    print("转换失败：", e)
```

输出：

```python
转换失败： invalid literal for int() with base 10: 'abc'
```

这样更利于排查问题 🔍

# 14. `else`：没出错时执行 🎉

很多人第一次学异常时不知道 `else`。

格式：

```python
try:
    可能出错的代码
except:
    出错时执行
else:
    没出错时执行
```

例子：

```python
try:
    num = int("123")
except ValueError:
    print("转换失败")
else:
    print("转换成功，结果是", num)
```

输出：

```python
转换成功，结果是 123
```

---

## 再看一个失败情况

```python
try:
    num = int("abc")
except ValueError:
    print("转换失败")
else:
    print("转换成功，结果是", num)
```

输出：

```python
转换失败
```

因为出错了，所以 `else` 不执行。

---

# 15. `finally`：不管出不出错都执行 🔒

这个也很重要。

格式：

```python
try:
    ...
except:
    ...
finally:
    ...
```

例子：

```python
try:
    print(10 / 0)
except ZeroDivisionError:
    print("除数不能为0")
finally:
    print("程序结束")
```

输出：

```python
除数不能为0
程序结束
```

---

## `finally` 有什么用？

最常见的用途是：

* 关闭文件
* 释放资源
* 收尾操作

也就是说：

> **不管成功还是失败，最后都得做的事，放 finally 里**

---

# 16. try-except-else-finally 完整结构 🏗️

```python
try:
    print("开始执行")
    num = int("123")
except ValueError as e:
    print("发生异常：", e)
else:
    print("没有异常，结果是：", num)
finally:
    print("这句一定会执行")
```

输出：

```python
开始执行
没有异常，结果是： 123
这句一定会执行
```

# 25. `except Exception as e` 是什么？🤨

这个很常见：

```python
try:
    ...
except Exception as e:
    print("发生异常：", e)
```

它表示：

> **捕获大多数常见异常**

因为很多错误都继承自 `Exception`。

这种写法在实际开发中很常用，特别是你不确定具体会出什么错的时候。

---

# 26. 但也不能乱用 `except Exception` ⚠️

虽然方便，但不要把所有代码都粗暴包起来。

比如：

```python
try:
    一大堆几十行代码
except Exception:
    print("出错了")
```

这样会导致：

* 不知道哪一行错
* 不知道为什么错
* 容易掩盖 bug

更好的做法是：

> **只包可能出错的那一小段代码**
> **尽量捕获明确的异常类型**

# 28. 更好的写法 ✅

```python

```

或者：

```python
try:

```

# 29. 异常不是让你“遮住错误”，而是让你“处理错误” 🧠

这是一个非常重要的观念。

很多初学者以为：

> 用异常处理，就是为了不让程序报错

不对。

真正目的是：

> **让程序在发生错误时，按你预期的方式处理，而不是直接崩掉。**

所以异常处理不是“掩耳盗铃”，而是“有计划地应对问题”。

# 30. 文件操作里的异常处理 📂

这是非常常见的应用场景。

```python
try:
    f = open("data.txt", "r", encoding="utf-8")
    content = f.read()
    print(content)
except FileNotFoundError:
    print("文件不存在")
finally:
    print("程序结束")
```

如果文件不存在，就不会直接崩。

---

# 31. 更好的文件写法：`with open(...)` ✨

虽然 `finally` 可以关闭文件，但 Python 更推荐：

```python
try:
    with open("data.txt", "r", encoding="utf-8") as f:
        content = f.read()
        print(content)
except FileNotFoundError:
    print("文件不存在")
```

因为 `with` 会自动帮你关闭文件。

你可以先记住：

> 文件操作里，`with` 很常用
> 异常处理负责应对打开失败、读取失败等情况

# 34. 异常在函数里怎么传递？📦

看例子：

```python
def test():
    print(10 / 0)

test()
```

`test()` 里面出了错，如果函数内部不处理，异常会往外传。

也就是说：

> **函数内部没处理，调用它的地方就会收到异常**

所以你可以在外面处理：

```python
def test():
    print(10 / 0)

try:
    test()
except ZeroDivisionError:
    print("捕获到了除零错误")
```

---

# 35. 这就叫“异常传播” 🌊

异常会一层一层往外冒，直到：

* 被某个 `except` 捕获
* 或者最终没人处理，程序崩掉

# 38. 一个完整实战：银行取钱 💰

```python
class Account:
    def __init__(self, owner, balance):
        self.owner = owner
        self.balance = balance

    def withdraw(self, money):
        if money <= 0:
            raise ValueError("取款金额必须大于0")
        if money > self.balance:
            raise ValueError("余额不足")
        self.balance -= money
        return self.balance


account = Account("张三", 1000)

try:
    left_money = account.withdraw(1200)
except ValueError as e:
    print("取款失败：", e)
else:
    print("取款成功，剩余余额：", left_money)
finally:
    print("操作结束")
```

# 40. 什么时候该用异常，什么时候不用？🧭

这个问题很重要。

---

## 适合用异常的情况

* 文件可能不存在
* 用户输入可能非法
* 转换类型可能失败
* 网络请求可能失败
* 调用外部资源可能失败

---

## 不适合滥用异常的情况

比如你明明能提前判断：

```python
if b != 0:
    print(a / b)
else:
    print("除数不能为0")
```

这种直接判断也很好，不一定非得靠异常。

所以更准确地说：

> **能正常判断的情况，用 if 判断**
> **不容易完全提前避免的问题，用异常处理**

---

# 41. 异常处理和 if 判断的关系 🧠

这俩不是敌人，而是配合关系。

例如：

## 业务规则判断

```python
if age < 0:
    print("年龄不合法")
```

## 运行时风险处理

```python
try:
    age = int(input("请输入年龄："))
except ValueError:
    print("请输入数字")
```

前者是逻辑判断，后者是运行时异常处理。

## 练习5

```python
class Account:
    def __init__(self, balance):
        self.balance = balance

    def withdraw(self, money):
        if money > self.balance:
            raise ValueError("余额不足")
        self.balance -= money
        print("取款成功，余额：", self.balance)

try:
    acc = Account(100)
    acc.withdraw(200)
except ValueError as e:
    print("操作失败：", e)
```

# 46. 最后做个大总结 🎯

Python 异常你现在可以这样整体理解：

## 异常是什么

程序运行时发生的错误

## 为什么要处理异常

不让程序直接崩掉，而是按预期处理错误

## 最核心的语法

```python
try:
    ...
except:
    ...
```

## 更完整的语法

```python
try:
    ...
except 异常类型 as e:
    ...
else:
    ...
finally:
    ...
```

## 主动制造异常

```python
raise ValueError("错误信息")
```
