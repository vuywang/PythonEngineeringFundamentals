# 1. 先说人话：类到底是什么？🤔

学类之前，你先记一句最重要的话：

> **类是模板，对象是根据模板创建出来的具体东西。**

比如：

* “学生” 是一个类别
* “张三这个学生” 是一个具体对象
* “小红这个学生” 也是一个具体对象

再比如 🚗

* “汽车” 是类
* 你家那辆白色汽车，是对象
* 别人家那辆黑色汽车，也是对象

所以可以先这样理解：

## 类

描述一类事物“有什么特征、能做什么事”

## 对象

这一类事物中的“某一个具体实例”

# 2. 为什么要学类？📦

前面你学函数时，已经能写代码了。
那为什么还要学类呢？

因为当程序越来越大时，光靠函数不够方便。
你会发现很多数据和行为，是属于“同一个东西”的。

比如学生系统里：

一个学生有这些数据：

* 姓名
* 年龄
* 分数

一个学生还会做这些事：

* 自我介绍
* 学习
* 显示成绩

这时候，把这些东西放在一起就更自然。
这就是类的用处 💡

也就是说：

> **类就是把“数据”和“操作这些数据的方法”放在一起管理。**

# 3. 最简单的类怎么写？🧱

Python 定义类用 `class`

```python

    pass
```

这里：

* `class`：定义类
* `Student`：类名
* `pass`：先占位置，表示暂时什么都不写

这个类现在已经定义好了。

你可以理解成：

> 按照 Student 这个模板，造了一个具体的学生对象 s1

# 9. `self` 是什么？这点很关键 🧠

很多初学者最容易卡在这里。

你先不要把 `self` 想得太神秘。
你可以先把它理解成：

> **self 就是“当前这个对象自己”**

比如：

```python
class Student:
    def __init__(self, name):
        self.name = name
```

假设：

```python
s1 = Student("张三")
```

这里的 `self`，其实就是 `s1` 这个对象本身。

所以：

```python
self.name = name
```

意思就是：

> 把传进来的 `name`，保存到“当前对象自己的 name 属性”里

也就是相当于：

```python
s1.name = "张三"
```

# 10. 为什么一定要写 `self`？🤨

因为类里的方法，要知道自己在操作“哪个对象”。

比如：

```python
class Student:
    def __init__(self, name):
        self.name = name
```

创建两个对象：

```python
s1 = Student("张三")
s2 = Student("李四")
```

如果没有 `self`，Python 就不知道：

* 这个 name 是给 `s1` 存
* 还是给 `s2` 存

有了 `self`，就能明确：

* `s1` 调用时，`self` 就是 `s1`
* `s2` 调用时，`self` 就是 `s2`

这样每个对象就有自己的数据了 🌟

# 17. 修改类属性要小心 ⚠️

```python
class Student:
    school = "第一中学"

    def __init__(self, name):
        self.name = name
```

如果你这样改：

```python
Student.school = "第二中学"
```

所有对象看到的 `school` 都会变。

但如果你写：

```python
s1.school = "第三中学"
```

这通常是在 `s1` 身上新建了一个同名对象属性，不是改类属性。

看例子：

```python
class Student:
    school = "第一中学"

    def __init__(self, name):
        self.name = name

s1 = Student("张三")
s2 = Student("李四")

s1.school = "第三中学"

print(s1.school)
print(s2.school)
print(Student.school)
```

输出：

```python
第三中学
第一中学
第一中学
```

说明：

* `s1.school` 变成了自己的对象属性
* `s2.school` 还是读类属性
* 类本身没变

# 19. 方法里还能修改对象属性 🔄

类的一个重要作用，就是让对象状态变化。

```python
class BankAccount:
    def __init__(self, owner, balance):
        self.owner = owner
        self.balance = balance

    def deposit(self, money):
        self.balance += money
        print(f"存款成功，当前余额：{self.balance}")

    def withdraw(self, money):
        if money > self.balance:
            print("余额不足")
        else:
            self.balance -= money
            print(f"取款成功，当前余额：{self.balance}")
```

使用：

```python
account = BankAccount("张三", 1000)
account.deposit(500)
account.withdraw(300)
account.withdraw(2000)
```

输出：

```python
存款成功，当前余额：1500
取款成功，当前余额：1200
余额不足
```

这里你就能看出：

> 类不仅能保存数据，还能通过方法改变数据

这就是“对象”的感觉了 💰✨

# 20. `__str__`：让对象打印得更好看 🖨️

输出：


# 24. 私有属性和私有方法 🔒

Python 没有特别强的“绝对私有”，但有一种约定式写法。

如果你不想让外部直接乱访问某个属性，可以在前面加两个下划线：

```python
class Person:
    def __init__(self, name, age):
        self.name = name
        self.__age = age
```

这里的 `__age` 可以理解成“内部使用的属性”。

使用：

```python
p = Person("张三", 18)
print(p.name)
```

这没问题。

但这样通常不行：

```python
print(p.__age)
```

会报错。

实际开发里，这更多是一种约定，而不是死锁。

# 26. getter 和 setter 是什么？🛎️

有时候你不希望别人直接改属性，而是想“通过方法”去改。

比如年龄不能是负数。

```python
class Person:
    def __init__(self, name, age):
        self.name = name
        self.__age = age

    def get_age(self):
        return self.__age

    def set_age(self, age):
        if age < 0:
            print("年龄不能为负数")
        else:
            self.__age = age
```

使用：

```python
p = Person("张三", 18)
print(p.get_age())

p.set_age(20)
print(p.get_age())

p.set_age(-5)
```

输出：

```python
18
20
年龄不能为负数
```

这就是：

* `get_age()`：获取值
* `set_age()`：设置值

# 27. 更 Python 的写法：`@property` ✨

Python 里经常不用传统的 `get_xxx()` / `set_xxx()` 风格，而是用 `@property`

这个你先认识一下：

```python
class Person:
    def __init__(self, age):
        self.__age = age

    @property
    def age(self):
        return self.__age

    @age.setter
    def age(self, value):
        if value < 0:
            print("年龄不能为负数")
        else:
            self.__age = value
```

使用：

```python
p = Person(18)
print(p.age)

p.age = 20
print(p.age)

p.age = -3
```

这里看起来像直接访问属性，但背后其实调用了方法。

初学阶段先知道这是“更高级、更常见的属性管理方式”就够了 🌈

# 28. 继承是什么？👨‍👩‍👧‍👦

继承是类里非常重要的一部分。

比如：

* 狗 和 猫 都属于动物
* 动物都有名字、会吃东西
* 狗会汪汪叫
* 猫会喵喵叫

那就可以写一个“父类” Animal，再让 Dog、Cat 继承它。

```python
class Animal:
    def __init__(self, name):
        self.name = name

    def eat(self):
        print(f"{self.name}正在吃东西")
```

然后写子类：

```python
class Dog(Animal):
    def bark(self):
        print(f"{self.name}正在汪汪叫")
```

使用：

```python
d = Dog("旺财")
d.eat()
d.bark()
```

输出：

```python
旺财正在吃东西
旺财正在汪汪叫
```

# 29. 继承有什么好处？🎁

因为公共部分不用重复写。

如果没有继承，你可能要在 Dog、Cat 里都写：

* `name`
* `eat()`

这样重复。

有了继承，就能把公共部分放父类里，子类只写自己的特色功能。

这叫：

> **代码复用**

# 32. `super()` 是什么？🚀

如果子类想在“保留父类逻辑”的基础上，再加自己的东西，就可以用 `super()`。

```python
class Animal:
    def __init__(self, name):
        self.name = name

class Dog(Animal):
    def __init__(self, name, age):
        super().__init__(name)
        self.age = age
```

这里：

```python
super().__init__(name)
```

意思是：

> 调用父类的 `__init__`，把 `name` 初始化好

然后子类再补自己的：

```python
self.age = age
```

使用：

```python
d = Dog("旺财", 3)
print(d.name)
print(d.age)
```

输出：

```python
旺财
3
```

---

# 33. 为什么要用 `super()`？🤔

因为如果父类已经写好了公共初始化逻辑，子类就别重复写了。

不用 `super()` 的话，你可能得手动写：

```python
self.name = name
```

但如果父类初始化逻辑很复杂，重复写就麻烦。

所以：

> **super() 就是“借用父类已有功能”**

# 34. 多态是什么？🎭

“多态”这个词看起来很吓人，其实意思很简单。

看例子：

```python
class Animal:
    def speak(self):
        print("动物发出声音")

class Dog(Animal):
    def speak(self):
        print("狗在汪汪叫")

class Cat(Animal):
    def speak(self):
        print("猫在喵喵叫")
```

再写个函数：

```python
def make_animal_speak(animal):
    animal.speak()
```

调用：

```python
make_animal_speak(Dog())
make_animal_speak(Cat())
```

输出：

```python
狗在汪汪叫
猫在喵喵叫
```

这里你会发现：

同一个函数 `make_animal_speak()`，传不同对象进去，会表现出不同结果。

这就是多态 🌟

简单理解：

> **同样的调用方式，面对不同对象，表现不同**

比如：

* Student("张三", 18)
* Dog("旺财")
* Car("比亚迪", "白色")

类像“图纸”，对象像“照图纸造出来的具体东西”。

# 36. 初学者最常见的错误 🚨

## 错误1：忘记写 `self`

```python
class Student:
    def say_hello():
        print("你好")
```

这通常不对，因为方法第一个参数要写 `self`。

正确：

```python
class Student:
    def say_hello(self):
        print("你好")
```

---

## 错误2：把 `name` 和 `self.name` 搞混

```python
class Student:
    def __init__(self, name):
        name = name
```

这没有把数据存到对象里。

正确：

```python
class Student:
    def __init__(self, name):
        self.name = name
```

左边 `self.name` 是对象属性。
右边 `name` 是传进来的参数。

---

## 错误3：类名和对象名分不清

```python
Student.name = "张三"
```

这通常不是你想要的，因为 `name` 一般是对象属性，不是类属性。

更常见的是：

```python
s1 = Student("张三")
print(s1.name)
```

---

## 错误4：创建对象时参数不匹配

```python
class Student:
    def __init__(self, name, age):
        self.name = name
        self.age = age

s1 = Student("张三")
```

会报错，因为少传了一个 `age`

# 37. 来一个完整小案例：学生类 🎓

```python
class Student:
    school = "第一中学"

    def __init__(self, name, age, score):
        self.name = name
        self.age = age
        self.score = score

    def introduce(self):
        print(f"我叫{self.name}，今年{self.age}岁，来自{self.school}")

    def study(self, subject):
        print(f"{self.name}正在学习{subject}")

    def exam(self):
        print(f"{self.name}这次考试得了{self.score}分")

    def __str__(self):
        return f"Student(name={self.name}, age={self.age}, score={self.score})"
```

使用：

```python
s1 = Student("张三", 18, 95)
s2 = Student("李四", 19, 88)

s1.introduce()
s1.study("Python")
s1.exam()

print(s1)
print(s2)
```

输出：

```python
我叫张三，今年18岁，来自第一中学
张三正在学习Python
张三这次考试得了95分
Student(name=张三, age=18, score=95)
Student(name=李四, age=19, score=88)
```

这个例子已经把很多核心知识串起来了：

* 类
* 对象
* `__init__`
* `self`
* 对象属性
* 类属性
* 方法
* `__str__`

# 38. 再来一个更像真实开发的小案例：购物车 🛒

```python
class CartItem:
    def __init__(self, name, price, count):
        self.name = name
        self.price = price
        self.count = count

    def get_total_price(self):
        return self.price * self.count

    def show(self):
        print(f"商品：{self.name}，单价：{self.price}，数量：{self.count}，小计：{self.get_total_price()}")
```

使用：

```python
item1 = CartItem("苹果", 5, 3)
item2 = CartItem("牛奶", 12, 2)

item1.show()
item2.show()
```

输出：

```python
商品：苹果，单价：5，数量：3，小计：15
商品：牛奶，单价：12，数量：2，小计：24
```

类特别适合表示“一个具体的业务对象”。

# 45. 最后送你一个超短公式版 🪄

你以后写类，脑子里可以按这个模板来：

```python
class 类名:
    类属性

    def __init__(self, 参数):
        self.对象属性 = 参数

    def 方法名(self):
        使用 self.对象属性
```

比如：

```python
class Student:
    school = "第一中学"

    def __init__(self, name):
        self.name = name

    def say_hello(self):
        print(f"你好，我是{self.name}")
```
